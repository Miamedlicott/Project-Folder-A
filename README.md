Responsive Website Project
---
---
Overview
-
This project involved turning my high-fidelity prototype into a responsive website using HTML5 and CSS3. The goal was to create a website that works across desktop, tablet and mobile screen sizes while keeping the design visually consistent and easy to use.
The website was built using semantic HTML, external CSS and responsive design techniques including Flexbox, CSS Grid and media queries. The project reflects both the visual direction established in the prototype stage and the practical considerations required during front-end development.
---
Design Process
-
I started by creating low-fidelity wireframes to plan the layout and structure of the website. The prototype acted as a visual guide during development and helped maintain consistency throughout the build process.  These helped me organise the content and think about how users would move through the site.
After this, I created a high-fidelity prototype using a free website builder called Webador that included colours, typography, spacing and imagery. Using this as a guideline, the final website closely follows the prototype, although some layout changes were made during development to improve responsiveness and usability on smaller screens. Some aspects were also adapted to cater to my personal abilities in coding.
Webador high-fidelity prototype: https://temp-aowxckvdqyzhmqygzpgu.webadorsite.com/ 
---
Development Process
-
The website was developed using semantic HTML5 elements such as `<header>`, `<nav>`, `<main>` and `<footer>` to create a clear and accessible structure.
CSS3 was used for all styling and layout implementation. Flexbox was primarily used for navigation alignment and smaller layout components, while CSS Grid was used to create responsive card layouts and scalable content sections. Media queries were implemented to adapt the website across multiple screen sizes including mobile, tablet and desktop resolutions.
Some techniques used in the project included: Semantic HTML5, CSS Grid, Flexbox, Media queries, CSS variables, and Hover effects.
---
Accessibility Considerations
-
Accessibility was considered throughout both the design and development stages of the project. Semantic HTML was used to improve screen reader compatibility and document structure. Images included alt text where needed and the website was tested across different screen sizes to ensure content remained readable and easy to navigate.
Colour contrast was also considered to maintain readability and visual clarity. Interactive elements such as buttons and navigation links were designed with hover states and sufficient spacing to improve usability across devices.
---
Challenges and Problem Solving
-
One challenge during development was making layouts work consistently across different screen sizes. Some sections looked good on desktop but needed adjustments on mobile devices. To solve this, I changed spacing, grid layouts and flexbox settings using media queries. I also tested the site regularly using a browser developer tool Codepen to check responsiveness.
Another challenge involved balancing visual design with performance and maintainability. Initially, some design ideas included more complex layout structures and effects, however these were simplified to create cleaner and more efficient code.
The most challenging part of the code was getting the footer to adapt to a smaller screen size without the contents overlapping.
Through this process, I developed a stronger understanding of responsive workflows and the importance of iterative testing during front-end development.
---
Reflection
-
Overall, I am happy with the final result because the website successfully reflects the original prototype and functions well across multiple devices. 
One strength of the project was the use of responsive layouts and organised code structure. I also improved my understanding of Flexbox, CSS Grid and responsive workflows throughout the development process. 
If I continued developing the project, I would improve the adaptability and accessibility further and add more advanced layouts and interactions.
This project helped me build confidence in front-end development and responsive web design.
